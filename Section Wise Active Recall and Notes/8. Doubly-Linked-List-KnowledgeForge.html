Doubly Linked List

1. Why a Doubly Linked List?

💡 Think of a singly linked list like a one-way street — you can only drive forward. A doubly linked list is a two-way street — each node knows both its neighbour ahead and its neighbour behind.

A singly linked list has a painful problem: to delete the last node, you must walk the entire list just to find the node before tail — because you can't go backwards. A doubly linked list solves this by giving every node a prev pointer in addition to the usual next pointer.

  • Singly linked list — each node stores: element + next
  • Doubly linked list — each node stores: element + next + prev

Node Class in C#

csharp
public class Node
{
    public int element;   // the data stored in this node
    public Node next;     // reference to the node AHEAD
    public Node prev;     // reference to the node BEHIND ← the new part!

    public Node(int e, Node n, Node p)
    {
        element = e;
        next    = n;
        prev    = p;
    }
}

2. addLast — Insert at End

💡 Creates a new node and stitches it onto the tail. Both the old tail's next and the new node's prev must be updated — that's the doubly-linked twist.
csharp
public void addLast(int e)
{
    Node newest = new Node(e, null, null);  // Step 1: create node
    if (IsEmpty)
    {
        head = newest;                       // Step 2a: empty list
        tail = newest;
    }
    else
    {
        tail.next    = newest;               // Step 3: old tail links forward
        newest.prev  = tail;                 // Step 4: new node links backward ← KEY
        tail         = newest;               // Step 5: advance tail
    }
    Size++;
}
⏱ Time: O(1) 📦 Space: O(1)

All operations are direct pointer assignments — no traversal needed.

3. addFirst — Insert at Beginning

💡 Mirror image of addLast. The new node goes before head. Update newest.next = head then head.prev = newest, then move head.
csharp
public void addFirst(int e)
{
    Node newest = new Node(e, null, null);
    if (IsEmpty)
    {
        head = newest;
        tail = newest;
    }
    else
    {
        newest.next  = head;      // Step 3: new node links forward to old head
        head.prev    = newest;    // Step 4: old head links backward to new node ← KEY
        head         = newest;    // Step 5: advance head pointer
    }
    Size++;
}
⏱ Time: O(1) 📦 Space: O(1)

4. addAnywhere — Insert at Any Position

💡 Walk to the node just before the target position. Then do a 4-way pointer swap to insert the new node between p and p.next.
⚠️ Order matters. Set newest.next and p.next.prev BEFORE changing p.next — otherwise you lose the reference to the node after p.
csharp
public void addAnywhere(int e, int position)
{
    if (position <= 0 || position >= Size)
    {
        Console.WriteLine("Invalid position. Use addFirst for 0, addLast for end.");
        return;
    }

    Node newest = new Node(e, null, null);
    Node p = head;

    for (int i = 1; i < position - 1; i++)   // traverse to node BEFORE target
        p = p.next;

    // 4-way pointer swap:
    newest.next  = p.next;      // Step A: new node → next node
    p.next.prev  = newest;      // Step B: next node ← new node (backward)
    p.next       = newest;      // Step C: prev node → new node
    newest.prev  = p;           // Step D: new node ← prev node (backward)

    Size++;
}
⏱ Time: O(n) — traversal to position 📦 Space: O(1)

5. removeFirst — Delete from Beginning

💡 Move head to head.next, then set the new head's prev to null — cutting the backward link to the deleted node.
csharp
public int removeFirst()
{
    if (IsEmpty)
    {
        Console.WriteLine("List is Empty.");
        return -1;
    }

    int e    = head.element;     // save value to return
    head     = head.next;        // advance head to next node
    Size--;

    if (IsEmpty)
    {
        tail = null;             // list is now empty — clear tail too
        head = null;
    }
    else
    {
        head.prev = null;        // ← KEY: sever backward link to deleted node
    }

    return e;
}
⏱ Time: O(1) 📦 Space: O(1)

6. removeLast — Delete from End

💡 This is the BIG win of doubly linked lists. Instead of traversing the whole list to find the second-last node, you just use tail.prev directly — instant O(1) access.
csharp
public int removeLast()
{
    if (IsEmpty)
    {
        Console.WriteLine("List is Empty.");
        return -1;
    }

    int e   = tail.element;   // save value to return
    tail    = tail.prev;      // ← KEY: jump directly to second-last node
    tail.next = null;          // sever forward link to deleted node
    Size--;

    if (IsEmpty)
    {
        head = null;
        tail = null;
    }

    return e;
}
⏱ Time: O(1) ← major advantage over singly linked! 📦 Space: O(1)

7. removeAnywhere — Delete at Any Position

💡 Walk to the node just before target. Then bypass the target node by redirecting both p.next and p.next.next.prev around it.
csharp
public int removeAnywhere(int position)
{
    if (position <= 0 || position >= Size)
    {
        Console.WriteLine("Use removeFirst or removeLast for ends.");
        return -1;
    }

    Node p = head;
    for (int i = 1; i < position - 1; i++)    // traverse to node BEFORE target
        p = p.next;

    int e    = p.next.element;    // save deleted element
    p.next   = p.next.next;       // bypass target node (forward link)
    p.next.prev = p;              // ← KEY: fix backward link of successor node

    Size--;
    return e;
}
⏱ Time: O(n) — traversal to position 📦 Space: O(1)

8. Display / Traversal

💡 Same as singly linked list — start at head, walk via next until null. The prev pointers enable backward traversal too (tail → head).
csharp
// Forward traversal (head → tail)
public void display()
{
    Node p = head;
    while (p != null)
    {
        Console.Write(p.element + ", ");
        p = p.next;
    }
    Console.WriteLine();
}

// Backward traversal (tail → head) — bonus power of doubly linked!
public void displayReverse()
{
    Node p = tail;
    while (p != null)
    {
        Console.Write(p.element + ", ");
        p = p.prev;
    }
    Console.WriteLine();
}
⏱ Time: O(n) 📦 Space: O(1)

Q01DEF What is a doubly linked list and how is its node structure different from a singly linked list?
Answer
A doubly linked list is a sequence of nodes where each node holds three things: element (the data), next (reference to the node ahead), and prev (reference to the node behind). A singly linked list only has element + next — it can't look backwards.
Example: Node(17) in doubly linked: element=17, next→Node(85), prev→Node(42)
Q02WHY Why is removeLast O(1) in a doubly linked list but O(n) in a singly linked list?
Answer
To delete the last node, you need to update the second-last node's next to null. In a singly linked list, you can only move forward — so you must traverse the whole list (O(n)) to reach the node before tail. In a doubly linked list, tail.prev gives instant access to that node — no traversal needed, so it's O(1).
Code: tail = tail.prev; tail.next = null; — two lines, no loop

Key: Set newest.next and head.prev BEFORE moving head. Order matters.
Q04GOTCHA In addAnywhere, why must you set newest.next and p.next.prev BEFORE setting p.next = newest?
Answer
If you set p.next = newest first, you overwrite the only reference to the node that was originally after p. That node becomes unreachable — you've lost everything from that point forward. Always save the reference to the next node (via newest.next = p.next) before changing p.next.
Rule: Read before overwrite. Steps A and B (set newest.next, fix successor.prev) must complete before step C (p.next = newest).
Q05SEQUENCE List the exact sequence of pointer updates to insert a new node between node p and node p.next in a doubly linked list.
Answer
Step A: newest.next = p.next — new node points forward to successor
Step B: p.next.prev = newest — successor points backward to new node
Step C: p.next = newest — predecessor points forward to new node
Step D: newest.prev = p — new node points backward to predecessor
A and B must come before C. D can be last.

Q07COMPARE Compare the time complexity of all operations between a singly and doubly linked list. Which operations differ and why?
Answer
Most operations are the same. The one key difference:
removeLast: Singly = O(n) (must traverse to find second-last). Doubly = O(1) (tail.prev is direct).
Everything else — addFirst, addLast, removeFirst — is O(1) in both. addAnywhere and removeAnywhere are O(n) in both (traversal required).
Trade-off: Doubly linked list costs one extra pointer per node (more memory) but gains O(1) removeLast.

Q09GOTCHA What happens if you forget to set head.prev = null in removeFirst? What bug does this cause?
Answer
The new head's prev still points to the deleted node. That deleted node is now unreachable from the list (nothing points to it), but it still has a reference from the new head. This is a memory leak — the garbage collector may not collect the deleted node because a pointer to it still exists. It can also cause logical bugs if you ever traverse backwards from head.
Fix: Always add head.prev = null; after advancing head in removeFirst.

Answer

2. List is not empty, so skip empty branch
3. tail.next = newest — Node(17)'s next now points to Node(99)
4. newest.prev = tail — Node(99)'s prev points back to Node(17)
5. tail = newest — tail reference moves to Node(99)
6. Size++ — size becomes 3

Q11DEF What does "link hopping" mean in the context of linked list traversal?
Answer
Link hopping is the process of moving from one node to the next by following the next (or prev) pointer, rather than jumping to a memory address directly. Each hop follows a reference stored in the current node — like following a chain of directions one step at a time until you reach null.
Code: p = p.next; is one hop forward.
Q12WHY Why should you never directly manipulate the head pointer during traversal? What do you use instead?
Answer
If you use head as your traversal pointer and advance it (head = head.next), you permanently lose the reference to the start of the list. There's no way to get back — the entire list before your current position becomes inaccessible. Always create a separate temporary pointer (Node p = head) and advance that while keeping head untouched.
Q13APPLY Given a doubly linked list [10 ↔ 20 ↔ 30 ↔ 40], write the pointer updates to remove the node at position 2 (value 20).

Q14COMPARE When would you choose a doubly linked list over a singly linked list? When would singly be better?
Answer
Choose doubly when: You need O(1) deletions at both ends (head and tail), backward traversal, implementing a deque, or building a browser history (forward/backward navigation).

Choose singly when: Memory is very tight (doubly uses one extra pointer per node), you only ever traverse forward, or the list is append-only (stacks, queues with one-way flow).
Q15GOTCHA What edge case must you handle in addLast and addFirst when the list is empty?
Answer
When the list is empty, both head and tail must point to the same new node. If you skip this and only update one of them, tail (or head) will remain null, making either the end or beginning of the list unreachable. This causes NullReferenceException on the very next operation.
Code: if (IsEmpty) { head = newest; tail = newest; }

Q17WHY Why does a doubly linked list need more memory than a singly linked list? Is there ever a scenario where this extra cost is not worth it?
Answer
Each node in a doubly linked list stores an extra prev reference (typically 4–8 bytes, same size as next). For a list of n nodes, that's n extra pointer-sized allocations. If you have a list of millions of nodes, this doubles the pointer overhead. Not worth it if: you only ever traverse forward, never delete from the tail, and memory is constrained (embedded systems, high-frequency data pipelines).

3. tail.next = null — sever forward link to deleted node
4. Size--
5. Return e
Q19APPLY An interviewer asks: "How would you implement a browser's back/forward navigation using a doubly linked list?" Sketch the approach.
Answer
Each page URL is a node. A current pointer tracks where you are.
Visit new page: Insert a new node after current, truncate everything after it (new branch), advance current.
Back: current = current.prev (if prev != null)
Forward: current = current.next (if next != null)

The prev pointer enables O(1) back navigation — exactly what makes doubly linked lists the right choice here.
Q20GOTCHA In removeAnywhere, after doing p.next = p.next.next, you must also do p.next.prev = p. What breaks if you skip this?
Answer
If you skip p.next.prev = p, the successor node's prev still points to the deleted node. If you later traverse backwards from anywhere after that point, you'll pass through the deleted node — which should no longer exist. You also lose the ability to correctly use backward traversal or backward-dependent operations from that region of the list.
Rule: Every doubly-linked pointer update is a pair — fix both directions, always.
Q21COMPARE How does a doubly linked list differ from an array in terms of memory layout and access patterns?
Answer
Array: Contiguous memory — all elements are adjacent. Index access is O(1) because you can compute an address directly (base + index × size). Insertion/deletion in the middle requires shifting elements: O(n).

Doubly Linked List: Nodes scattered anywhere in memory — connected only by pointers. Random access is O(n) because you must hop from head. Insertion/deletion (once positioned) is O(1) — just pointer reassignment.
Q22APPLY What is the overall space complexity of a doubly linked list with n nodes, and what contributes to it?
Answer
Space: O(n)
Each of the n nodes stores 3 fields: element (the data value), next (one reference/pointer), prev (one reference/pointer). So total memory = n × (sizeof element + sizeof next + sizeof prev). The head, tail, and size variables add O(1) overhead. Total: O(n).
vs Singly: Singly linked = n × 2 fields. Doubly = n × 3 fields — 50% more pointer overhead.
