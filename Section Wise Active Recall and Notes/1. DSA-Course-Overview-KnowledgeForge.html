Data Structures & Algorithms in C#
Course Overview & Foundations

🗄️ What is Data & Why Store It Efficiently?

💡 Everything on your computer — documents, photos, videos, programs — is data. A data structure is just a smart way to organise that data so your program can find and use it fast.
  • Data = useful information stored in computer memory
  • Examples: a document file, a photo, an audio clip, a software program
  • Data structure = a systematic method to organise and access data efficiently
  • Without a structure, your app would have to scan everything every time — like searching a library with no shelves
📌 Every program that does something real uses both a data structure (to store info) and an algorithm (to act on it). These two always come as a pair.

The Two-Component Model of Problem Solving

  • Component 1 — Data Structures: store information associated with the problem
  • Component 2 — Algorithms: the step-by-step method that solves the problem
  • Efficiency is the critical design factor in both — a slow algorithm or poorly chosen structure can make software unusable

📦 Categories of Data Structures

💡 Think of linear structures like a line of people — each element connects to the next. Non-linear structures are like a family tree — one element can connect to many others.

Linear Data Structures

  • Stack — last in, first out (LIFO). Like a stack of plates.
  • Queue — first in, first out (FIFO). Like a checkout line.
  • Linked List — chain of nodes, each pointing to the next. Dynamic size.

Non-Linear Data Structures

  • Trees — hierarchical. Parent nodes branch to children. Used in file systems, databases.
  • Heaps — specialised tree. Always keeps the min or max at the root. Powers priority queues.
  • Graphs — nodes connected by edges (no strict hierarchy). Used in maps, social networks.

⚙️ Why Learn Algorithms?

💡 An algorithm is just a recipe — a finite list of steps that takes an input and produces an output. Without a good recipe, even the best ingredients (data) produce a bad meal (slow app).
  • Algorithm = a set of instructions that solves a problem in a finite number of steps
  • Essential for real-world applications requiring systematic approaches
  • Small problems are solvable by hand — large problems are not
⚠️ Faster hardware does NOT fix a bad algorithm. Data volume grows faster than CPU speed. An O(n²) algorithm that works for 1,000 items can become unusable at 1,000,000 items — even on a faster machine.

The Navigation Example

  • Small road network → you can find the best path manually
  • City-wide road network → impossible without a computer and an efficient pathfinding algorithm (e.g. Dijkstra's)
  • Same principle applies everywhere: search engines, social feeds, recommendations

The Data Growth Problem

  • 15 years ago: Facebook had thousands of users → brute force worked
  • Today: billions of users → the same brute-force approach would take days
  • Hardware improvements slow down; data volumes keep accelerating
  • Conclusion: Algorithm efficiency is a permanent engineering concern, not a one-time optimisation
📌 You need a systematic way to compare algorithms BEFORE writing code — not after. That's what algorithm analysis (Big O, time complexity, space complexity) gives you.

What Algorithm Analysis Lets You Do

  • Predict execution speed and memory use before implementation
  • Compare two solutions on paper — pick the better one
  • Guarantee an algorithm actually finishes (termination proof)
  • Understand the theoretical limits of what's possible

🛠️ Visual Studio Setup for C#

💡 Visual Studio is the IDE (Integrated Development Environment) — it's the workshop where you write, run, and debug C# code. Think of it like Word, but for writing programs instead of documents.

Creating a New Project

Three ways to start — all lead to the same dialog:

  • File menu → New → Project
  • New Project button on the menu bar
  • New Project link on the start page
📌 For this course: always select Console Application under Visual C#. It gives you a plain terminal window to run and test your data structure code without UI overhead.

Project Configuration Steps

  1. Open the New Project dialog
  2. Select Visual C# in the leftmost column
  3. Choose Console Application
  4. Enter a project name (e.g. LearnDSAlgorithms)
  5. Choose save location (default or custom)
  6. Solution name auto-fills from project name — leave it
  7. Click OK → Visual Studio generates the starter code

The Auto-Generated Program Structure

csharp 
using System;

namespace LearnDSAlgorithms
{
    class Program
    {
        // Entry point — execution starts here
        static void Main(string[] args)
        {
            Console.WriteLine("Hello World");
            Console.ReadKey(); // pause so you can see the output
        }
    }
}
  • using System; — imports the System namespace so you can use Console, etc.
  • namespace — a container that gives your class a unique scope
  • static void Main(string[] args) — the program entry point; execution always starts here
  • Console.ReadKey() — pauses the console window so output doesn't disappear immediately

Q01 DEF In one sentence, what is a data structure?
Answer
A data structure is a systematic method of organising and storing data in computer memory so that programs can access and manipulate it efficiently.
Example: A list of student names stored as a linked list — each name points to the next — so you can traverse, insert, or delete without shifting everything.
Q02 WHY Why can't you solve every problem by just throwing faster hardware at it?
Answer
Data volume grows much faster than CPU speed. An inefficient algorithm that works for 1,000 items can take exponentially longer at 1,000,000 items — no hardware improvement covers that gap. Algorithm efficiency is a software-side constraint, not a hardware one.
Example: Facebook scaled from thousands to billions of users. Brute-force search that ran in seconds for thousands would take days for billions — regardless of processor speed.
Q03 COMPARE What is the difference between a linear and a non-linear data structure? Give one example of each.
Answer
  • Linear: elements are arranged in a single sequence — each element has at most one predecessor and one successor. Example: Queue
  • Non-linear: elements have branching or mesh relationships — one element can connect to many others. Example: Tree
Analogy: A queue is a straight line of people. A tree is a company org chart — one CEO, many branches below.
Q04 DEF What is an algorithm, and what does "finite" mean in its definition?
Answer
An algorithm is a set of instructions that solves a problem in a finite number of steps. Finite means it must terminate — it can't run forever. An algorithm that loops indefinitely is not a valid algorithm.
Example: Dijkstra's shortest-path algorithm terminates once every node has been visited — it doesn't run forever.
Q05 SEQUENCE List the course topics in the correct learning order, from first to last.
Answer
  1. Setup & prerequisites (Visual Studio, C# basics)
  2. Algorithm analysis — time/space complexity, Big O
  3. Recursion and recursive algorithm analysis
  4. Searching and sorting algorithms
  5. Linear data structures — linked lists, stacks, queues, deques
  6. Non-linear structures — binary trees, BSTs, heaps, hashing, graphs, BFS/DFS
Why this order: Each phase depends on the previous. You can't analyse a tree traversal without understanding recursion first.

Q07 GOTCHA You run your C# program and a build error dialog appears. What should you click — Yes or No — and why?
Answer
Click No. Clicking Yes runs the previous successful build — not your current code. You'll see old output and incorrectly think the error is gone. Click No, find the error in the Error List, fix it, then run again.
Common error: A missing semicolon triggers this dialog. The error list will show the exact line number.
Q08 WHY Why does this course teach algorithm analysis before teaching individual algorithms?
Answer
Analysis gives you the language and tools to compare algorithms before writing code. Without it you'd have to implement every candidate solution and benchmark it — which is slow, error-prone, and impossible for large inputs. Big O notation lets you compare on paper first.
Example: You can say O(n log n) sort > O(n²) sort without writing a single line of either.
Q09 COMPARE Compare a Stack and a Queue: how does each structure determine which element to remove first?
Answer
  • Stack (LIFO) — Last In, First Out. The most recently added item is removed first. Like a stack of plates — you take from the top.
  • Queue (FIFO) — First In, First Out. The oldest item is removed first. Like a checkout line — the first person in line is served first.
Real use: Stack → undo/redo history. Queue → print job queue, HTTP request handling.

Q14 WHY Why is recursion taught before trees and graphs in this course?
Answer
Tree traversal (in-order, pre-order, post-order) and graph search (DFS) are inherently recursive operations. If you don't understand recursion — base cases, call stack, return values — you can't understand why these algorithms work. Recursion is the prerequisite for most non-linear DS operations.

Q19 GOTCHA A classmate says: "My algorithm works perfectly for 500 items, so we don't need to worry about efficiency." What's wrong with this reasoning?
Answer
Performance for small inputs doesn't predict performance at scale. An O(n²) algorithm at 500 items might run in milliseconds — but at 500,000 items it's 1,000,000× slower in theoretical terms, making it completely unusable. Efficiency must be analysed as a function of input size, not tested only at one small size.
Numbers: If 500 items takes 0.01s with O(n²), then 500,000 items would take ~10,000 seconds. That's 2.7 hours.
Q20 COMPARE What is the difference between BFS and DFS in terms of how they explore a graph?
Answer
  • BFS (Breadth-First Search) — visits all neighbours of the current node before going deeper. Explores level by level. Uses a Queue internally.
  • DFS (Depth-First Search) — goes as deep as possible down one branch before backtracking. Uses a Stack (or recursion) internally.
Analogy: BFS = exploring a building floor by floor. DFS = entering every room on one floor fully before moving to the next floor.
