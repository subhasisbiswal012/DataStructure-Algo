1. What is a Tree?

💡 Think of a company org chart — CEO at the top, managers in the middle, employees at the bottom. That hierarchy is a tree. Everyone except the CEO has exactly one boss, and some people manage others below them.

A tree is a nonlinear, hierarchical Abstract Data Type (ADT) consisting of nodes connected by edges. Unlike arrays or linked lists (sequential, before/after), trees arrange data in levels where some elements are above others.

  • Linear structures (array, linked list, stack, queue) — elements in a line, before/after relationships.
  • Nonlinear structures (trees, graphs) — elements in levels, above/below relationships.
  • For n nodes, a tree always has exactly n − 1 edges. Always.
  • A tree can be empty (zero nodes) — valid state, just like an empty stack.
📌 Iron rule: A tree with n nodes always has exactly n − 1 edges. No exceptions.

2. Tree Terminology

Root, Parent & Child

  • Root node — the only node with no parent. Sits at the top. Every tree has exactly one root.
  • Parent node — a node that has one or more children directly below it.
  • Child node — a node directly below another node (its parent).

Siblings, Leaf & Internal Nodes

  • Siblings — nodes sharing the same parent. Marketing, Sales, Purchase, Manufacturing are all siblings (parent = Company). Domestic and International are siblings (parent = Sales).
  • Leaf / External node — node with no children. End of a branch. (Marketing, Purchase, Domestic, Phones, TVs, India, America, UK)
  • Internal / Non-leaf node — node with at least one child. (Sales, Manufacturing, International, America)

Edge, Path, Subtree & Forest

  • Edge — a directed link between a parent and its child. In a tree with n nodes there are exactly n−1 edges.
  • Path — a sequence of nodes where every consecutive pair is connected by an edge. Paths run from root to leaf.
  • Subtree — any node plus all of its descendants. A single leaf node is also a valid subtree.
  • Forest — a collection of trees. Remove the root of a tree → you get a forest of subtrees.

Level vs Height

💡 Both measure depth, but count differently. Level counts nodes in the path to root (starts at 1). Height counts edges in the path to root (starts at 0).
  • Level — number of nodes on path from the node to root (including both endpoints). Root = Level 1.
  • Height — number of edges on path from the node to root. Root = Height 0.
  • Relationship: Level = Height + 1
  • Tree height = maximum level − 1

Node Degree & Tree Degree

  • Node degree — the count of children a node has. Leaf nodes always have degree 0.
  • Tree degree — the maximum degree found among all nodes in the tree. The root doesn't always have the highest degree.

3. Binary Tree — Definition & Formula

💡 "Binary" = 2. A binary tree is a tree where each node can have at most 2 children — named left child and right child.
  • Every node has 0, 1, or 2 children — never 3 or more.
  • The two children are labeled left child and right child — order matters.
  • Alternative definition: a tree with degree ≤ 2.

4. Types of Binary Trees

Proper / Strict Binary Tree

📌 Every node must have exactly 0 or 2 children. Any node with exactly 1 child disqualifies the entire tree.
✅ IS Proper

All nodes have 0 or 2 children. No node has exactly 1. ✓

❌ NOT Proper

Node C has exactly 1 child. Degree 1 anywhere → NOT proper. ✗

Full Binary Tree

📌 Two conditions BOTH required: (1) Every internal node has exactly 2 children. (2) All leaf nodes are at the same level. Formula: must have exactly 2(h+1) − 1 nodes.
✅ IS Full (h=2, 7 nodes)

7 nodes = 2^(2+1)−1 ✓ All internals have 2 children ✓ All leaves at level 3 ✓

❌ NOT Full (gap in leaves)

C has 1 child (deg=1). F is a leaf at level 2 while D,E are at level 3. Fails both conditions. ✗

Complete Binary Tree

📌 All levels fully filled except possibly the last. The last level is filled left to right with no gaps. Number nodes 1,2,3… left-to-right level by level — there must be no missing numbers.
✅ IS Complete

Positions 1–6 filled left-to-right. C has no right child — that's fine, last level fills left first. ✓

❌ NOT Complete (gap)

Position 5 (B's right child) is missing but position 6 (C's right child) exists. Gap = NOT complete. ✗

Skewed Binary Trees

ℹ️ Every node has exactly one child, creating a linear chain. Technically a binary tree, but it performs like a linked list — O(n) search instead of O(log n).
Left Skewed

Every node only has a left child. Height = n−1. Worst case for most operations.

Right Skewed

Every node only has a right child. Same O(n) problem. Happens when inserting sorted data into a BST.

6. Array Representation of Binary Trees

💡 Store a binary tree in a plain array using index math instead of pointers. Number nodes level-by-level left-to-right starting at index 1 — and the parent-child relationship falls out of simple arithmetic.
  • Array indices start at 1 (not 0) — makes the formulas clean.
  • Root is always at index 1.
  • Level-order numbering: each level numbered left to right.
  • Array size required = 2(h+1) − 1 for a tree of height h.

7. Linked Representation

💡 Instead of index math, each node carries explicit memory addresses to its left and right children. Nodes can be anywhere in RAM — the links stitch them together.

Each node contains three fields: a left pointer, the data value, and a right pointer.

8. Binary Tree Traversal

💡 Traversal = visiting every node in the tree exactly once in a defined order. Trees are nonlinear so there are multiple valid orderings — each useful for different tasks.
  • Depth-First (DFS) — go deep before wide. Pre-order, In-order, Post-order are all DFS. Use recursion or a Stack.
  • Breadth-First (BFS) — go wide before deep. Level-order is BFS. Requires a Queue.

Pre-order: Root → Left → Right

📌 Visit the current node first, then recurse on the left subtree, then recurse on the right subtree.

In-order: Left → Root → Right

📌 Recurse into the left subtree all the way down, then visit the root, then the right subtree. For a BST, this gives nodes in sorted ascending order.

Post-order: Left → Right → Root

📌 Visit the left subtree, then the right subtree, and only then the root — which is always visited last.

Level-order (BFS): Top to Bottom, Left to Right

📌 Visit all nodes at level 1, then all at level 2, then level 3, and so on. Within each level, visit left to right. Needs a Queue to implement.

9. Memory Shortcut — a + b Trick

💡 Remember the simple math expression a + b. The position of the operator (+) tells you which traversal it maps to — and where the root is visited.
📌 Pre = Prefix = +ab (root first). In = Infix = a+b (root middle). Post = Postfix = ab+ (root last). This pattern works for any binary tree.

Q01DEFWhat is a tree data structure and how does it differ from linear data structures like arrays and linked lists?
Answer
A tree is a nonlinear, hierarchical ADT consisting of nodes connected by edges, with parent-child relationships between levels. Linear structures (array, linked list, stack, queue) arrange elements sequentially — every element has a before and after. Trees arrange elements hierarchically — some elements are above others (parent), some below (children). Trees enable faster algorithm implementations for many operations and are modelled after family trees (parent, child, sibling). For n nodes, a tree always has exactly n−1 edges.

Answer

Sibling: Nodes sharing same parent — Marketing, Sales, Purchase, Manufacturing are siblings (parent = Company). Leaf: Node with no children, degree 0 — Marketing, Purchase, Domestic, India. Internal: Node with at least one child — Sales, Manufacturing, International.
Q03DEFWhat is the difference between Level and Height of a node? What is their exact mathematical relationship?
Answer
Level: Number of nodes on the path from the node to the root, including both endpoints. Root = Level 1. Height: Number of edges on the path from the node to the root. Root = Height 0. Relationship: Level = Height + 1. Tree height = maximum level − 1.
Example: A node at Level 4 has 3 edges to root → Height 3.
Q04DEFDefine node degree and tree degree. Does the root always have the highest degree in a tree?
Answer
Node degree: Number of children a node has. Leaf nodes always have degree 0. Tree degree: Maximum node degree among all nodes in the tree — the tree's branching factor. No — the root doesn't always have the highest degree. If Company has 2 children but International has 3, tree degree = 3 (from International, not root).
Q05DEFDefine edge, path, subtree, and forest in tree terminology.
Answer
Edge: A directed connection between a parent node and its child. A tree with n nodes has exactly n−1 edges. Path: Sequence of nodes where every consecutive pair is connected by an edge (e.g., Company → Sales → International). Subtree: Any node plus all its descendants. A single leaf is also a valid subtree. Forest: Collection of disjoint trees. Removing a tree's root creates a forest of independent subtrees.
Q06DEFWhat is a Binary Tree? What values can a node's degree take? What is the maximum nodes formula?
Answer
A binary tree is a tree where every node has at most two children — specifically named left child and right child. Node degree can be 0, 1, or 2. It can also be defined as a tree with degree ≤ 2. Max nodes formula: A binary tree of height h can contain at most 2^(h+1) − 1 nodes. Height 0 → 1 node. Height 1 → 3 nodes. Height 2 → 7 nodes. Height 3 → 15 nodes.
Q07DEFWhat is a Proper (Strict) Binary Tree? What single rule defines it, and what disqualifies a tree from being proper?
Answer
A proper (strict) binary tree is one where every node has exactly 0 or 2 children — never exactly 1. A single node with degree 1 anywhere in the entire tree disqualifies it. Leaf nodes (degree 0) are always fine. To verify: check every node — if any has exactly 1 child, the tree is NOT proper.
Quick test: Scan all nodes. Any degree-1 node → fail immediately.
Q08DEFWhat is a Full Binary Tree? State both required conditions and the node-count formula.
Answer
A full binary tree requires BOTH: (1) Every internal node has exactly 2 children. (2) All leaf nodes are at the same level. Missing either condition = not full. A full binary tree of height h has exactly 2^(h+1)−1 nodes. Equivalently, if a binary tree of height h has exactly 2^(h+1)−1 nodes, it is a full binary tree. Full → also proper AND complete.
Q09DEFWhat is a Complete Binary Tree? What must be true about the last level? How do you detect an incomplete tree?
Answer
A complete binary tree has all levels fully filled except possibly the last, which must be filled left to right with no gaps. To detect: number all nodes 1,2,3... level by level left-to-right. If any number is missing while a higher number exists → NOT complete. A missing rightmost node at the last level is fine; a missing middle node with nodes to its right is not.
Gotcha: Position 5 missing but position 6 exists = gap = NOT complete.

Q11COMPARECompare array representation vs linked representation of a binary tree — memory, access speed, and ideal use cases.
Answer
Array: Pre-allocates 2^(h+1)−1 slots. O(1) access by index. No pointer overhead. Wastes space for sparse/skewed trees. Ideal for complete/full trees and heaps. Linked: Each node = [left_ptr | data | right_ptr]. Grows dynamically. Works for any shape tree. O(n) traversal to reach a node. No wasted slots but 2 pointer fields per node. Choose array when tree is complete/full. Choose linked for general trees.

Q13APPLYWhat is the a+b memory shortcut for remembering traversal orders? Map it to all three depth-first traversals.
Answer
Use the math expression a + b. The position of the operator (+) = position of the root in the traversal:
• Pre-order = Prefix → +ab → operator (root) BEFORE operands → Root First
• In-order = Infix → a+b → operator (root) IN BETWEEN → Root Middle
• Post-order = Postfix → ab+ → operator (root) AFTER operands → Root Last
Root = operator. Left child = a. Right child = b.
Q14COMPAREIs every Full binary tree also Complete? Is every Complete binary tree also Full? Justify both answers.
Answer
Full → Complete: YES. A full binary tree has all levels completely filled (every level, including the last). This satisfies the complete tree requirement — no gaps, last level filled L→R. Complete → Full: NO. A complete tree's last level may be only partially filled (not all 2^h nodes present). So it has fewer than 2^(h+1)−1 total nodes — not full. Example: 6-node height-2 complete tree is NOT full (needs 7 nodes to be full).

── Real-Life Interview Questions (Q16–Q45) ──
Q16DEFWhat is a Binary Search Tree (BST)? How does it differ from a regular binary tree?
Answer
A BST is a binary tree with an ordering constraint: for every node, all values in its left subtree are less and all values in its right subtree are greater. A plain binary tree has no such constraint — it just limits each node to ≤ 2 children. BST enables O(log n) search/insert/delete in balanced state. In-order traversal of a BST always yields sorted ascending output.
Q17APPLYHow do you find the height of a binary tree? Write the recursive logic in C#.
Answer
csharp
int Height(Node root) {
    if (root == null) return -1;  // empty tree
    int leftH  = Height(root.left);
    int rightH = Height(root.right);
    return 1 + Math.Max(leftH, rightH);
}
Time: O(n) — visits every node once. Space: O(h) for recursion stack.
Q18APPLYHow do you check if a binary tree is balanced? What does "balanced" mean?
Answer
Balanced = for every node, the height difference between left and right subtrees is at most 1. Recursively compute heights. If |leftH − rightH| > 1 at any node → not balanced. Optimised approach: return −1 as a sentinel when an unbalanced subtree is detected, to short-circuit further recursion. Time O(n).
Examples: AVL trees and Red-Black trees maintain balance automatically. Skewed tree = worst case unbalanced.
Q19APPLYHow do you find the Lowest Common Ancestor (LCA) of two nodes in a binary tree?
Answer
Recurse from root. Base case: if null or current node equals either target, return current node. Get LCA results from left and right subtrees. If both return non-null, current node is the LCA (targets are on different sides). If only one side returns non-null, that result propagates up. Time O(n).
Key insight: The first node where the two targets "split" into different subtrees is the LCA.
Q20APPLYHow do you serialize and deserialize a binary tree? Which traversal is best, and why?
Answer
Use pre-order traversal. Serialize: output each node's value; output "null" for missing children. This captures the complete structure. Deserialize: read the pre-order sequence left-to-right; create each node and recursively build its left then right subtrees. Pre-order works because the root is always first — you can immediately create it and then reconstruct its subtrees.
Example: Tree A(B,C) → "A,B,null,null,C,null,null" → deserialises back to exact same tree.
Q21APPLYHow do you find the diameter (longest path between any two nodes) of a binary tree?
Answer
The diameter may or may not pass through the root. For each node: candidate diameter = height(left) + height(right) + 2. Track the global maximum. Use a recursive helper that returns height of each subtree while updating a maxDiameter variable. Time O(n) — single DFS pass.
Key insight: The longest path passes through the node that maximises leftH + rightH — not necessarily the root.
Q22APPLYHow do you check if two binary trees are structurally identical?
Answer
Recurse simultaneously on both trees. Base cases: (1) Both null → true. (2) One null, one not → false. (3) Values differ → false. Recursive case: both left subtrees are identical AND both right subtrees are identical. All three checks must pass. Time O(min(n1, n2)).
Q23APPLYHow do you mirror (invert) a binary tree? Write the C# logic.
Answer
csharp
Node Mirror(Node root) {
    if (root == null) return null;
    // Swap children
    (root.left, root.right) = (root.right, root.left);
    Mirror(root.left);
    Mirror(root.right);
    return root;
}
Swap left and right pointers at every node, then recurse. Time O(n).
Q24APPLYHow do you find all root-to-leaf paths in a binary tree?
Answer
DFS with backtracking. Maintain a current path list. At each node: add it to the path. If it's a leaf (both children null), record the path as a complete root-to-leaf path. Otherwise, recurse on left and right children. After recursion, remove the current node (backtrack). This systematically explores all root-to-leaf paths.
Q25APPLYHow do you find the level with the maximum number of nodes in a binary tree?
Answer
Level-order BFS with a Queue. At the start of each level, snapshot the queue size — that's the node count for that level. Track the maximum. The snapshot before dequeuing each level's nodes tells you exactly how many nodes are on that level.
Pattern: int levelSize = queue.Count; before the inner for-loop.
Q26WHYWhy is in-order traversal of a BST important? What does it always produce?
Answer
In-order traversal (Left → Root → Right) of a BST always produces values in sorted ascending order. This is because BST's ordering property (left subtree < node < right subtree) aligns perfectly with in-order's visit pattern — smallest values first, then the root (middle), then larger values. This enables O(n) sorted extraction from a BST with no extra sorting step.
Q27COMPAREWhat is the difference between DFS and BFS traversal? Which tree traversals are DFS and which are BFS?
Answer
DFS (Depth-First): Goes deep before wide — explores one branch completely before backtracking. Pre-order, In-order, Post-order are all DFS. Implemented with recursion or an explicit Stack. BFS (Breadth-First): Goes wide before deep — explores all nodes at current level before going deeper. Level-order is BFS. Requires a Queue. BFS finds shortest paths in unweighted trees/graphs; DFS is better for tree operations like copy, delete, and expression evaluation.
Q28APPLYHow do you count the number of leaf nodes in a binary tree?
Answer
Recurse: if node is null return 0. If node is a leaf (both children null), return 1. Otherwise return CountLeaves(left) + CountLeaves(right). Single DFS pass, O(n). Every node is visited exactly once.
Q29GOTCHACan you uniquely reconstruct a binary tree from just one traversal (pre-order alone, or in-order alone)? What combination is needed?
Answer
No. A single traversal cannot uniquely identify a binary tree — multiple different trees can produce the same pre-order or in-order sequence. You need two traversals: Pre-order + In-order, OR Post-order + In-order. In-order is essential in both combinations because it identifies which nodes belong to the left vs right subtrees. Pre-order + Post-order alone is not sufficient for trees with single-child nodes.
Q30APPLYHow do you print a binary tree in zigzag (spiral) order?
Answer
Level-order BFS with a direction flag. Even levels: left to right. Odd levels: right to left. Use two stacks or a deque: for left-to-right levels enqueue children left-then-right; for right-to-left levels enqueue right-then-left. Alternate the flag after each level completes.
Q31APPLYHow do you find the maximum path sum in a binary tree (path can start and end at any node)?
Answer
For each node: candidate = node.val + max(leftGain, 0) + max(rightGain, 0). Update global maximum. Return to parent: node.val + max(leftGain, rightGain, 0) — only one branch goes upward. Use max(gain, 0) to exclude negative-sum subtrees. Single DFS pass, O(n). Track maxSum as a global variable or ref parameter.
Q32COMPAREWhat is the difference between a Heap and a BST? When do you use each?
Answer
Heap: Complete binary tree. Parent ≥ children (max-heap) or parent ≤ children (min-heap). No ordering between siblings. O(1) peek at max/min, O(log n) insert/delete. Used for priority queues and heapsort. BST: Left < node < right strictly. O(log n) search/insert/delete when balanced. O(n) worst case (skewed). Used for sorted data, ordered maps, range queries.
Q33APPLYHow do you validate whether a binary tree is a valid BST? What's the common mistake?
Answer
Use min/max bounds. Recurse: IsValid(node, min, max). Each node must satisfy min < node.val < max. Left subtree: upper bound = node.val. Right subtree: lower bound = node.val. Initial call: IsValid(root, int.MinValue, int.MaxValue). Common mistake: Only comparing a node with its immediate children — this misses cases where a distant ancestor's constraint is violated (e.g., a node in the right subtree smaller than the root).
Q34APPLYHow do you find the kth smallest element in a BST efficiently?
Answer
In-order traversal of a BST gives ascending sorted order. Count nodes as you visit them in-order. When the count reaches k, that node is the answer. Use an iterative in-order with a Stack and stop after k dequeues to avoid traversing the whole tree unnecessarily. Time O(h + k) where h is height.
Why? In-order of BST = sorted ascending. kth visit = kth smallest value.
Q35WHYWhy is post-order the correct traversal for deleting a binary tree? Why can't you use pre-order?
Answer
Post-order visits children BEFORE the parent, so you delete leaves first and then their parents. No parent is freed while its children still hold memory. Pre-order visits the root first — deleting it loses the only references to its children, causing a memory leak in languages without GC, or orphaning subtrees. The rule: always free children before freeing the parent = post-order.
Q36APPLYWrite iterative in-order traversal of a binary tree in C# without recursion.
Answer
csharp
void InOrderIterative(Node root) {
    var stack = new Stack<Node>();
    Node curr = root;
    while (curr != null || stack.Count > 0) {
        // Go all the way left
        while (curr != null) {
            stack.Push(curr);
            curr = curr.left;
        }
        // Visit node, then go right
        curr = stack.Pop();
        Console.Write(curr.val + " ");
        curr = curr.right;
    }
}
The explicit Stack simulates the recursive call stack. Time O(n), Space O(h).
Q37APPLYHow do you check if a binary tree is symmetric (a mirror image of itself)?
Answer
Write helper IsMirror(left, right): both null → true. One null → false. Values differ → false. Then: IsMirror(left.left, right.right) AND IsMirror(left.right, right.left). Call IsMirror(root.left, root.right) from the main function. This checks that outer pairs and inner pairs mirror each other at every level.
Q38GOTCHAWhat is a skewed binary tree and why is it a severe performance problem?
Answer
A skewed binary tree has every node with only one child — all left (left-skewed) or all right (right-skewed). With n nodes, height = n−1 instead of log n. This degrades BST operations from O(log n) to O(n) — equivalent to a linked list. It happens when inserting already-sorted data into a BST (1,2,3,4,5 → every node goes right). Self-balancing trees (AVL, Red-Black) prevent this by performing rotations after inserts/deletes.
Q39APPLYHow do you find the sum of all nodes at depth k in a binary tree?
Answer
Recurse with a depth counter. When depth == k, return node.val. If depth > k or node is null, return 0. Otherwise return SumAtDepth(left, depth+1, k) + SumAtDepth(right, depth+1, k). Alternatively, use level-order BFS and stop when you reach level k, then sum all nodes dequeued in that iteration.
Q40APPLYReconstruct a binary tree from its post-order and in-order traversal sequences.
Answer
In post-order, the last element is always the root. Find this root in the in-order array — elements to its left are in the left subtree, elements to its right are in the right subtree. Count the left subtree size. Use that count to split the post-order array into left and right parts. Recurse on both halves. Use a HashMap for O(1) in-order index lookup. Total time O(n).
Q41COMPAREWhat is an AVL tree? How does it differ from a regular BST and why does it matter?
Answer
An AVL tree is a self-balancing BST. After every insert or delete, it checks the balance factor (|leftH − rightH|) for affected nodes. If any node has |balance| > 1, it performs rotations (LL, RR, LR, RL) to restore balance. This guarantees height stays O(log n) always, so search/insert/delete remain O(log n) in the worst case. A regular BST on sorted input becomes a right-skewed chain with O(n) height — all operations degrade to O(n).
Q42APPLYHow do you find the second largest element in a BST?
Answer
Two cases: (1) The largest node (rightmost) has a left subtree → second largest is the rightmost node of that left subtree. (2) The largest node has no left child → second largest is the largest node's parent. Traverse right to find the largest while keeping a pointer to its parent. Then handle the two cases. Time O(h).
Q43APPLYWrite level-order traversal that prints each level on a separate line, using a Queue in C#.
Answer
csharp
void LevelOrder(Node root) {
    if (root == null) return;
    var queue = new Queue<Node>();
    queue.Enqueue(root);
    while (queue.Count > 0) {
        int levelSize = queue.Count; // nodes at this level
        for (int i = 0; i < levelSize; i++) {
            Node node = queue.Dequeue();
            Console.Write(node.val + " ");
            if (node.left  != null) queue.Enqueue(node.left);
            if (node.right != null) queue.Enqueue(node.right);
        }
        Console.WriteLine(); // new line per level
    }
}
Q44COMPAREWhat is the time complexity of search in a balanced BST vs an unbalanced (skewed) BST? Why?
Answer
Balanced BST: Height = O(log n). Search = O(log n). At each comparison, half the remaining search space is eliminated — same as binary search on a sorted array. Skewed BST: Height = O(n). Search = O(n). Each comparison only eliminates one node. Degrades to a linear scan — equivalent to unsorted linked list traversal. This is why self-balancing trees exist.
Q45APPLYHow do you count total nodes in a complete binary tree in less than O(n) time?
Answer
For a complete binary tree, compute leftmost height (go left repeatedly) and rightmost height (go right repeatedly). If equal: left subtree is a perfect binary tree → left count = 2^h − 1, add root, recurse right only. If unequal: right subtree is a perfect tree one level shorter → right count = 2^(h-1) − 1, add root, recurse left only. Each step eliminates half the tree. Time O(log²n) instead of O(n).
