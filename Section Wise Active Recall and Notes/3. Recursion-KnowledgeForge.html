🔁 How Recursion Works

💡 A function that calls itself is recursive. Think of it like Russian dolls — each doll opens to reveal a smaller version, until you reach the smallest one (the base case) that doesn't open further.

Two Required Components

  • Base condition — the stopping rule. Without it, recursion runs forever (stack overflow).
  • Self-call — the function calls itself with a smaller input, moving toward the base case.

What Happens on Every Function Call

  1. Calling function pauses
  2. Control transfers to the called function
  3. Called function executes completely
  4. Control returns to the caller
  5. Remaining statements execute

Worked Example — Fun(n) prints n²

csharp
static void Fun(int n)
{
    if (n > 0)           // base condition: stop when n = 0
    {
        int k = n * n;
        Console.WriteLine(k);
        Fun(n - 1);      // recursive call — smaller input
    }
}
// Fun(4) → prints: 16, 9, 4, 1

⚖️ Iteration vs Recursion

💡 Iteration and recursion solve the same problems. Iteration uses a loop; recursion uses self-calls. The while condition becomes the if base condition.
🔄 Iterative
void CalcIterative(int n)
{
  while (n > 0)
  {
    int k = n * n;
    Console.WriteLine(k);
    n = n - 1;
  }
}
🔁 Recursive
void CalcRecursive(int n)
{
  if (n > 0)         // base condition
  {
    int k = n * n;
    Console.WriteLine(k);
    CalcRecursive(n - 1);
  }
}

⏱️ Time Complexity of Recursive Functions — Recurrence Relations

💡 You can't analyse a recursive function the same way as a loop. Instead, write a recurrence relation — a math equation that expresses T(n) in terms of T(n-1) — then solve it.

How to Write a Recurrence Relation

  1. Count every primitive operation in one call → that's the constant part
  2. Add the cost of the recursive call → T(n-1)
  3. State the base case T(0) or T(1)

Example 1 — T(n) = T(n-1) + 1 → O(n)

Recurrence
—T(n) = T(n-1) + 3, T(0) = 1
—Simplified (constants don't affect Big-O):
—T(n) = T(n-1) + 1
Successive Substitution
1T(n) = T(n-1) + 1
2T(n) = T(n-2) + 1 + 1 = T(n-2) + 2
3T(n) = T(n-3) + 3
kT(n) = T(n-k) + k
Reach Base Case — set n−k = 0 → k = n
—T(n) = T(0) + n = 1 + n
—T(n) = O(n)

Example 2 — T(n) = T(n-1) + n → O(n²)

Recurrence
—T(n) = T(n-1) + n, T(1) = 1
Expansion
1T(n) = T(n-1) + n
2T(n) = T(n-2) + (n-1) + n
3T(n) = T(n-3) + (n-2) + (n-1) + n
kT(n) = T(n-k) + (n-k+1) + … + n
Reach Base Case — set n−k = 1 → k = n-1
—T(n) = T(1) + 2 + 3 + … + n = 1 + Σᵢ₌₂ⁿ i
—Σᵢ₌₁ⁿ i = n(n+1)/2 → Σᵢ₌₂ⁿ i = n(n+1)/2 − 1
—T(n) = 1 + n(n+1)/2 − 1 = n(n+1)/2 = (n²+n)/2
—T(n) = O(n²)

🔽 Tail Recursion

💡 Tail recursion = the recursive call is the last statement. No work happens after the call returns. Think: do your job, then hand off.
csharp
void CalculateTail(int n)
{
    if (n > 0)
    {
        int k = n * n;
        Console.WriteLine(k);   // work BEFORE the call
        CalculateTail(n - 1);   // recursive call LAST ← tail
    }
}
// Output for n=4: 16, 9, 4, 1  (descending)
📌 Tail recursion can be converted to a loop automatically by the compiler (Tail Call Optimization / TCO). It is iteration-equivalent — no pending work accumulates on the stack.

🔼 Head Recursion

💡 Head recursion = the recursive call comes first, before any work. The function dives all the way to the base case, then does work on the way back up.
csharp
void CalculateHead(int n)
{
    if (n > 0)
    {
        CalculateHead(n - 1);   // recursive call FIRST ← head
        int k = n * n;
        Console.WriteLine(k);   // work AFTER the call returns
    }
}
// Output for n=4: 1, 4, 9, 16  (ascending — reversed!)

🌳 Tree Recursion

💡 Tree recursion = a function calls itself more than once per invocation. Each call spawns multiple children → the call structure fans out like a tree.
csharp
void Calculate(int n)
{
    if (n > 0)
    {
        Calculate(n - 1);        // first recursive call
        int k = n * n;
        Console.WriteLine(k);
        Calculate(n - 1);        // second recursive call
    }
}
// Calculate(3) → output: 1, 4, 1, 9, 1, 4, 1

Time Complexity of Tree Recursion

Recurrence (2 calls per level)
—T(n) = 2·T(n-1) + 1, T(0) = 1
Expansion
1T(n) = 2·T(n-1) + 1
2T(n) = 2·[2·T(n-2)+1] + 1 = 2²·T(n-2) + 2 + 1
3T(n) = 2³·T(n-3) + 2² + 2 + 1
kT(n) = 2ᵏ·T(n-k) + (2ᵏ - 1)
Set n - k = 0 → k = n
—T(n) = 2ⁿ·T(0) + (2ⁿ - 1) = 2ⁿ + 2ⁿ - 1 = 2ⁿ⁺¹ - 1
—T(n) = O(2ⁿ)

⚠️ Tree recursion is O(2ⁿ) — exponential. Avoid for large n without memoization. Classic example: naive Fibonacci. With memoization / DP, tree recursion collapses back to O(n).

🔀 Indirect Recursion

💡 Indirect recursion = functions call each other in a circle. A calls B, B calls A, and so on. No function calls itself directly — but the chain loops back.
csharp
void CalculateA(int n)
{
    if (n > 0)
    {
        Console.WriteLine("A(" + n + ")");
        CalculateB(n - 1);     // A calls B
    }
}

void CalculateB(int n)
{
    if (n > 1)
    {
        Console.WriteLine("B(" + n + ")");
        CalculateA(n - 1);     // B calls A
    }
}
// CalculateA(5) → A(5) B(4) A(3) B(2) A(1)
📌 Base conditions are mandatory in every participating function — not just one. Missing a base condition in either function causes infinite recursion and stack overflow.

Σ Sum of N Natural Numbers — Three Approaches

💡 Problem: compute 1 + 2 + 3 + … + n. Three solutions with very different complexity. Knowing all three shows when to prefer each.
csharp
// 1. Direct Formula — O(1) time, O(1) space
int SumFormula(int n) => n * (n + 1) / 2;

// 2. Iterative — O(n) time, O(1) space
int SumIterative(int n)
{
    int total = 0;
    for (int i = 1; i <= n; i++) total += i;
    return total;
}

// 3. Recursive (Head Recursion) — O(n) time, O(n) space
int SumRecursive(int n)
{
    if (n == 0) return 0;         // base case
    return SumRecursive(n - 1) + n; // head: recurse first, add on return
}
// SumRecursive(5) → 15

Recurrence for SumRecursive

Recurrence
—sum(n) = sum(n-1) + n, sum(0) = 0
—This is the same as Example 2 above → T(n) = O(n)
—T(n) = O(n)

! Factorial — Recursive and Iterative

💡 n! = 1 × 2 × 3 × … × n. Unlike sum, there's no O(1) formula, so iterative and recursive are both O(n).

Mathematical Definition

Definition
—0! = 1(special base case)
—n! = (n-1)! × n    for n > 0
—4! = 3! × 4 = 2! × 3 × 4 = 1 × 2 × 3 × 4 = 24
🔄 Iterative — O(n) time, O(1) space
int FactIterative(int n)
{
  int result = 1;
  for (int i = 1; i <= n; i++)
    result *= i;
  return result;
}
// FactIterative(5) → 120
🔁 Recursive — O(n) time, O(n) space
int FactRecursive(int n)
{
  if (n == 0) return 1;  // base
  return FactRecursive(n - 1) * n;
}
// FactRecursive(5) → 120

Answer

  • Self-call with reduced input — the function calls itself with a smaller version of the problem, converging toward the base case.

Q03WHY Recursion and iteration produce the same output. Why does recursion use more memory?
Answer
Each recursive call pushes a new stack frame onto the call stack. The frame holds local variables, parameters, and the return address. For n calls, you have n frames simultaneously in memory — O(n) stack space. Iteration uses a fixed number of variables — O(1) space.
Risk: Deep recursion (large n) can exhaust the call stack → StackOverflowException.

void F(int n) { if(n>0) { print(n); F(n-1); } }
Answer
Recurrence:
—T(n) = T(n-1) + 1, T(0) = 1
—T(n) = T(n-k) + k → at k=n: T(0)+n = 1+n
—T(n) = O(n)
One print + one comparison = constant work per call. n+1 calls total.
Q05DEF What is tail recursion? How does it differ from general recursion?
Answer
Tail recursion is when the recursive call is the last statement in the function — no computation occurs after the call returns. The function completes all its work before delegating to itself. In general recursion, there may be pending work after the recursive call returns (e.g., head recursion multiplies after returning).
Key benefit: Tail calls can be optimized by the compiler (TCO) — the current stack frame is reused instead of a new one being pushed, making it O(1) space like iteration.
Q06GOTCHA CalculateTail(4) outputs 16, 9, 4, 1. If you move the recursive call before the print, what does the output become and why?
Answer
Output becomes 1, 4, 9, 16 (ascending). Moving the call before the print converts it to head recursion. The function dives all the way to n=1 first, printing nothing. Then on the way back up, each returning frame executes its print — so the smallest value (1) prints first.
Rule: Tail = work before call = top-down order. Head = work after call = bottom-up order.

Q09DEF What is tree recursion and why is it exponential?
Answer
Tree recursion is when a function calls itself more than once per invocation. Each call spawns multiple children, which each spawn more children — the call count doubles (or multiplies) at every level. With 2 calls per level and n levels: total calls = 2⁰ + 2¹ + 2² + … + 2ⁿ = 2ⁿ⁺¹ - 1 → O(2ⁿ).
Classic example: Fibonacci(n) = Fibonacci(n-1) + Fibonacci(n-2) — two calls per invocation → O(2ⁿ) without memoization.
Q10APPLY Solve the tree recursion recurrence:

Answer
1T(n) = 2·T(n-1) + 1
2T(n) = 2²·T(n-2) + 2 + 1
kT(n) = 2ᵏ·T(n-k) + (2ᵏ - 1)
—Set n-k=0 → k=n
—T(n) = 2ⁿ·T(0) + 2ⁿ - 1 = 2ⁿ + 2ⁿ - 1 = 2ⁿ⁺¹ - 1
—T(n) = O(2ⁿ)
Q11GOTCHA Tree recursion with n=3 makes exactly 15 calls. How do you get that number without tracing every call?
Answer
Use the formula: Total calls = 2ⁿ⁺¹ - 1. For n=3: 2⁴ - 1 = 16 - 1 = 15. This comes from summing all nodes at each level of the binary call tree: 2⁰ + 2¹ + 2² + 2³ = 1 + 2 + 4 + 8 = 15.
Q12DEF What is indirect recursion? What is the minimum requirement for it to terminate?
Answer
Indirect recursion is when two or more functions call each other in a cycle — A calls B, B calls A, creating a loop without any function calling itself directly. For it to terminate, every participating function must have a base condition — not just one of them. If any function in the chain lacks a base condition, the chain will loop forever.

Q16WHY Why is tail recursion said to be equivalent to iteration?
Answer
In tail recursion, no work is pending after the recursive call returns — all computation is complete before the self-call. This means the current stack frame is no longer needed once the recursive call starts. A compiler with Tail Call Optimization (TCO) can reuse the same stack frame for every call, making the total stack usage O(1) — exactly like a while loop. The execution is identical to iteration.
Q17GOTCHA Is return Factorial(n-1) * n tail recursion? Why or why not?
Answer
No — it is not tail recursion. Although the recursive call appears last in the source code, the actual last operation is the multiplication (× n) which happens after the recursive call returns. There is pending work on the stack. This is head/general recursion, not tail recursion.
Tail version: use an accumulator — FactTail(n-1, acc * n) where the multiply happens before the call.
Q18COMPARE What is the output of CalculateTail(3) vs CalculateHead(3) (printing n²)? Explain the difference.

  • Head(3): 1, 4, 9 — calls first, prints on return, so the deepest frame (n=1) prints first → ascending order
Same values, opposite order. The only structural difference is where the recursive call sits relative to the print statement.

Q20WHY Why is tree recursion avoided in practice for problems like Fibonacci without optimization?
Answer
Tree recursion recomputes the same subproblems repeatedly. Fibonacci(5) calls Fibonacci(3) twice — and each of those calls Fibonacci(2) twice — leading to exponential redundant work. The total calls grow as O(2ⁿ). For n=40, that's over a trillion calls. Memoization (caching results) reduces this back to O(n) by ensuring each subproblem is computed only once.
Q21APPLY What is the recurrence relation and Big-O for a function that calls itself twice per level with constant extra work?
Answer
—T(n) = 2·T(n-1) + 1, T(0) = 1
—Solution: T(n) = 2ⁿ⁺¹ - 1
—T(n) = O(2ⁿ)
The branching factor of 2 causes exponential growth. Each level doubles the number of active calls.
Q22GOTCHA A function has two separate recursive calls (not nested). Someone says: "Two calls, so it's O(2n) = O(n)." What's wrong?
Answer
Wrong. O(2n) = O(n) applies when two separate loops run sequentially. Two recursive calls in the same function create tree recursion — each call spawns two more at the next level, leading to exponential growth: 1 → 2 → 4 → 8 → ... → 2ⁿ calls total. The complexity is O(2ⁿ), not O(n).

Q24SEQUENCE What are the steps of the successive substitution method for solving a recurrence?
Answer
  1. Write the recurrence: T(n) = T(n-1) + f(n)
  2. Substitute the relation for T(n-1), T(n-2), etc. to find a pattern
  3. Express the general form after k substitutions: T(n) = T(n-k) + g(k)
  4. Set n-k = base case value (0 or 1) → solve for k
  5. Substitute the base case T(0) or T(1) to get the closed form
  6. Apply Big-O — drop constants and lower-order terms
Q25WHY Why must the recursive call in a recursive function always use a smaller input?
Answer
The recursion must converge toward the base case. If each call uses the same or larger input, the base case is never reached — the function recurses infinitely and eventually crashes with a stack overflow. Using n-1, n/2, etc. guarantees that after a finite number of calls, the base condition triggers and recursion terminates.
